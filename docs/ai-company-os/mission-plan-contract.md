# AI Company OS: Mission Plan Contract

This document defines the standard schema contract for Mission Plans generated by the OS.

## Mission Plan Structure

A **Mission Plan** consists of top-level metadata identifying the source goal, planning mode, and safety posture, alongside an array of dependency-ordered missions.

### Top-Level Fields

| Field | Type | Description |
| ----- | ---- | ----------- |
| `plan_id` | `string` | Unique identifier for the plan (e.g. `plan_20260629_grow_os`). |
| `milestone` | `string` | OS milestone version (e.g., `1.0C`). |
| `source_goal` | `string` | High-level company goal text. |
| `goal_type` | `string` | Classification category (e.g., `company_ops`, `product_delivery`). |
| `planning_mode` | `string` | Planner mode: `static` or `dry_run`. |
| `created_by_agent` | `string` | Executive agent generating the plan (usually `ceo_agent`). |
| `owner_approval_required` | `boolean` | Whether owner approval is required before execution. |
| `dispatch_allowed` | `boolean` | Statically sets whether dispatching to queues is allowed. |
| `dispatch_blocked_reason` | `string` | Detailed reason if dispatch is blocked. |
| `safety_summary` | `string` | Summary of safety boundaries and blocked actions. |
| `missions` | `array` | List of mission objects. |

### Mission Fields

Each mission in the `missions` array must include:

| Field | Type | Description |
| ----- | ---- | ----------- |
| `mission_id` | `string` | Unique identifier for the mission (e.g. `m01_audit`). |
| `title` | `string` | Short human-readable title. |
| `mission_type` | `string` | Mapping identifier from `mission-types.json` (e.g. `REPO_AUDIT`). |
| `target_factory` | `string` | Destination factory (e.g. `ai_dev_factory`). |
| `target_capability` | `string` | Target capability ID from the registry (e.g. `dev_repo_audit`). |
| `owner_agent` | `string` | Responsible executive agent (e.g. `cto_agent`). |
| `required_inputs` | `array` | Input parameters required to start the task. |
| `expected_outputs` | `array` | Output artifacts expected on completion. |
| `safety_class` | `string` | Safety boundary category (e.g. `safe_local`, `publish_blocked`). |
| `blocked_actions` | `array` | List of actions blocked during execution. |
| `approval_required_actions` | `array` | Gates requiring owner approval token. |
| `depends_on` | `array` | IDs of previous missions that must finish first. |
| `status` | `string` | Mission lifecycle state (e.g. `planned`, `ready_for_dry_run`). |
| `dispatch_allowed` | `boolean` | True if this mission can be sent to the queue. |
| `dispatch_blocked_reason` | `string` | Reason if dispatch is disabled. |
| `verifier_requirements` | `array` | List of verifier scripts needed to validate outputs. |
| `handoff_target` | `string` | Downstream agent or capability for handoff. |
